Маппинг ассоциаций (связи между таблицами)- механизм, который объединяет несколько таблиц по FORGEN KEY. 

1. В базе данных создаем поле, которое ссылается на другую таблицу(таблица company) по id другой таблицы(FORGEN KEY)
2. Создаем сущность для таблицы company
3. В ней указываем автогенерируемый id и поле name
4. В самой сущности user создаем поле с типом данных той сущности, которую мы создали (пункс2). Пример: private Company companyId;
5. В самой сущности, над полем companyId, ставим аннотации @ManyToOne and @JoinColum(name = "имяIdИзбазыданных")
   Пример:
   @ManyToOne
  
   private Company companyId; 

6. в методе main создаем через builder сущность company и сохраняем ее в базу данных и лишь потом сохраняем сущность User
7. в hibernate.comfig маппим сущность company


 @JoinColum(name = "company_id") <--- используется при маппинге ассоциаций
================================================================================================================================================
По умолчание, hibernate использует letft join, который сказывается на производительность запроса, чтобы изменить left join на inner join нужно использовать параметры @ManyToOne.
@ManyToOne(optional = false, fetch = FetchType.LAZY) 
optional = false       <--  Позволяет в запросе использовать inner join
fetch = FetchType.LAZY <--- Создает отдельный запрос на таблицу(в нашем примере на таблицу company)
