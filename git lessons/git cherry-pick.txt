Шаги: 
     1. Нам нужно перейти в ту ветку, в которую мы хотим вставить коммит из другой ветки 
        (У нас две ветки, master, developer. Мы хотим в ветку master вставить коммит из ветки developer)
         1.1 git checkout master
         1.2 git cherry-pick хеш_коммита 
         теперь в ветке master будет коммит из другой ветки(у взятого коммита будет другой хеш, формально в ветке master, появится новый коммит с новым хешем).
 
====================================================================================================================================================================================
Сначала переключитесь на ветку, в которую вы хотите вставить коммиты. В вашем случае, вы хотите вставить коммиты из ветки developer в ветку master:
 
# Переключитесь на ветку master
  git checkout master

# Для вставки одного коммита по его хешу
  git cherry-pick хеш_коммита (git cherry-pick хеш_коммита1 хеш_коммита2)
 
# Если вы хотите изменить сообщение коммита перед его вставкой, используйте --edit
  git cherry-pick --edit хеш_коммита

# Если вы хотите вставить коммиты, но не сразу фиксировать изменения, используйте --no-commit
git cherry-pick --no-commit хеш_коммита
После выполнения git cherry-pick коммит(ы) из ветки developer будут вставлены в ветку master с новыми хешами, так как они фактически созданы заново.

Если вы хотите, чтобы в сообщении коммита было указано, что он был взят с другой ветки, вы можете использовать опцию -x:
 
git cherry-pick -x хеш_коммита
Если вы хотите добавить подпись в сообщение коммита, указывая, кто сделал cherry-pick, вы можете использовать опцию --signoff:
shell
Copy code
git cherry-pick --signoff хеш_коммита
После выполнения этих шагов, ветка master будет содержать коммиты из ветки developer.
