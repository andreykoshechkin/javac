Централизованный подход (CVCS) -  это когда существует единый центральный репозиторий, где хранятся все версии проекта. Разработчики в свою очередь, скачивают свои копии проекта из центрального репозитория, 
				  работают с ними у себя на компьютере и затем отправляют изменения обратно в центральный репозиторий. 

Распределенный подход (DVCS)   -  это тот подход, который использует git и этот подход предпологает, что у каждого разработчика есть свой локальный репозиторий в котором лежит полностью история проекта. 
				  Разработчики могут совершать операции без необходимости постоянного подключения к центральному репоиторию.
				  Проще говоря, у проекта есть много копий и эти копии есть у каждого разработчика, поэтому, если центральный репозиторий сломается, то мы не потеряем весь проект
				  
Важным отличием DVCS от (CVCS),   является способность работать оффлайн и иметь всю историю проекта локально.
