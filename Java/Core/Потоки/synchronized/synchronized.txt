synchronized - Используется когда два потока пишут
Ключевое слово synchronized: Позволяет синхронизировать потоки тем самым убрать состояние гонки(race condition). Synchronized МОЖНО указывать только у метода. 

synchronized работает на каком-то объекте, java в фоновом режиме создает некий монитр и этот монитор можент принадлежать только одному потоку.
Именно по этой причине несколько потоков не могут выполнять один и тот же метод. Получается, что один поток ждет пока другой поток завершит действие.


synchronized блок - позволяет установить замок на маонитор, таким образом, одним монитором может пользоваться только один поток.
Если какая-то переменная статическая то мы синхронизируемся по всему классу

Когда мы не указываем объект для synchronized, мы синхронизируемся на объекте this.

synchronized блок - synchronized блок указывается в методе и он эквивалентен public synchronized void() за исключением, что в блоке нужно явно указать объект.
Syntaxis:

1) public synchronized void ИмяМетода()

2) public void ИмяМетода(){
 synchronized(this){
}
}
Оттичие synchronized блока от метода - synchronized блок позволяет синхронизировать какуюю-то часть кода.
Если мы хоти создать свой монитор, то он должен быть static final :  static  final Object obj = new Object();
