hashcode() и equals()		
equals - проверяет не по ссылке, а по значению
Для String and Integer методы hashcode() и equals() уже переопределенны,  а для объектов нет

hashcode() - этот метод, который на вход получает число случайно длины а на выходе возвращает число фиксированной длины(сам hashode) но так как, hashcode
возвращает int переменную, то может случиться колизия, именно по этой причине два разных объекта могу иметь одинаковый hashcode и 
в таком случае будет вызвае метод equals но если hachcode у двух объектов разный то эти объекты точно разные.


Когда мы сравниваем два объекта мы используем метод hashcode так как он работает быстрее чем equals и если объекты имеют разных hashcode то эти объекты точно
разные, если hashcode у объектов одинаковый, это означает, что либо случилась коллизия либо эти объекты действительно одинаковые.
