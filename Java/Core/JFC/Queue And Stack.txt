Queue(Очередь) - Это не синхранизованная структура данных, которая следует правилу FIFO (first in first out).
Классы который реализует интерфейс Queue, один из классов это  LinkedList (полный список: https://docs.oracle.com/javase/7/docs/api/java/util/Queue.html)
Один из классов который реализует Queue - ArrayBlockingQueue(Пишем длину очереди) 

Выбрасывают исключения:
T	
add(e) - добовляет элемент в queue			
remove()		
element()	

Возвращает специальное значение:
offer(e) - вставляет элемент, если это возможно, в противном случае возвращает false
poll() - удаляют и возвращают голову очереди
peek() - возвращают, но не удаляют заголовок очеред	


Stack(Класс) - это синхронизованная структура данных, которая представлена ввиде обратной очереди.Это класс который следует правилу LIFO
 Stack<Integer> stack = new Stack<>();

Методы:
push() - Помещает элемент в стек; 
pop() -  Удаляет элемент из стека при этом возвращает(выводит элемент на экран) элемент;
peek() - Получает элемент но не удаляет его.
empty() - Проверяет, пуст ли этот стек.

Deque(Вторичная очередь) - это интерфес который поддерживает вставку(или удаления) на обоих концах очереди

Iterable - это интерфейс, и все java колекции реализуют этот интерфейс. Он позволяет проходиться по нашим эелементам в цикле forEach. Он имеет один метод iterator
Iterator сначало указывает не на како-то элемент,  а на пространство в начале
