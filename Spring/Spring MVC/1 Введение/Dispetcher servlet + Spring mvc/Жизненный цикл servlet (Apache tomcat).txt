Клиент отправляет запрос, этот запрос проходит через coyote, далее происходит перенаправление запроса в контейнер сервлетов.


Coyote  - занимается тем, что открывает TCP соединение на том хосте на которым развернуто наше приложение.


Жиненный цикл сервлета состоит из 3 шагов:
1. Инициализация сервлета, которая срабатывает один раз на самый первый request. Сама инициализация сервлета так же происхожит в 3 шага:
   1. Загрузка servlet класса в JVM 
   2. Создание объекта servlet класса  
   3. и вызов метода init()
   После того, как сервлет был проинициализирован, он может принимать HTTP запросы.

2. Вызов метода service у servlet'a, это будет происходить до тех пор, пока мы не положем наше приложение.
   тут будет какая-то логина которая будет вызываться из метода serivice.

3. Последний шаг это вызов метода destroy. Этот метод вызываеться когда мы завершаем работу apache tomcat.
