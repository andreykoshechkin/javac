Примечение: Аннотацию @Autowired не принято ставить над полями! Их принято ставить именно над конструктором, но если мы решили ставить их над конструктором, то мы должны отказаться от xml конфигурации
потому что в случае xml, когда мы создаем бин, мы должны указывать конструктор, если мы конструктор не укажем, то должен быть конструктор без параметров, а так не должно, потому что мы хотим делать наши поля 
final, а значит у нас должен быть один конструктор для инициализации этих полей. 


@Component		# Эта аннотация, которая определяет класс как компонент, тем самым, позволяет спрингу автоматический находить этот класс по фильтру скана и регистрировать его в контексте как bean.
			  Но у аннотации @Component(@Service, @Repository, и @Controller) есть и не достаток, который заключается в том, что мы не сможем создавать несколько бинов одного типа. 
			
			  @Service, @Repository, и @Controller так же являются компонентом, данные аннотации нужны для логического разделения. 
 			  @Service    - ставится над слоем сервис.
			  @Repository - ставится над слоем repository(dao). 
 			  @Controller - ставится над слоем нашем контроллером.

  
   1. Если класс имеет только один конструктор, рекомендуется явно добавлять @Autowired, даже если это не является обязательным, для улучшения читаемости кода и явного выражения намерений.
   2. Если класс имеет несколько конструкторов, то также рекомендуется явно указывать @Autowired над одним из конструкторов, чтобы Spring знал, какой конструктор использовать для внедрения зависимостей.

<context:component-scan base-package="org.example.*"> 	# Позволяет искать классы, которые отмечены аннотацией @Component по указанному пути. Если класс подходит по условию фильтра, то будет создан 
							  beanDefinition на основании которого, будут созданы бины.


<context:component-scan					#  Внутри себя включает <context:annotation-config/>, тем самым добавляет бины, которые являются beanPostProcessor.
