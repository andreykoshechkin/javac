@Controller     - Это аннотация всего лишь компонент, которая просто указывает что GreetingController относится к слою Controller. Ставиться над классом
@RequestMapping - Предназначена для того, чтобы задать методам контроллера адреса, по которым они будут доступны на клиенте. На практике используеться для указания общего
                  префикса для всех методов.
                  Пример: Над классом ставим @RequestMapping(value = "/app") теперь адресс по которому мы будем переходить называеться http://localhost:8080/app/byeController
@GetMapping	- Замена @RequestMapping, которая сразу подразумевает использования get метода, @PostMetod для post метода и т.д

============================================================================================================================================================
Анноатции для работы с параметрами, header, coockie:
@RequestParam	 - Аннотация позволяет добавлять к url какие-то параметры. Spring сам преобразуем значение параметра в нужный тип данных и все что нам нужно сделать
		   это после @RequestParam указать тот тип данных, который мы хотим получить.
@RequestHeader	 - Аннотация которая дает возможность получить header страницы, в качестве параметров нужно передать mime type
@CookieValue	 - Позволяет получить coockie по заданному параметру

@SessionAttribute  - Используеться в тех случаях, когда нам нужно достать значение из сессии. (На курсе Http/Servlets req.getSession().setAttribute("user", user))
@SessionAttributeы - Используется тогда, когда мы хотим хранить ключи в сессии
