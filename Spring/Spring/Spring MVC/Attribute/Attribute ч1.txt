Повторение:
Attribute - нужны для того чтобы хранить какие-то значения в сессии. Attribute представляют из себя
	    ассотивный массив где Ключом явялется  String, а значением любой объект в том числе и java объект(объектDTO).

Виды attribute:
	1 - ServletContext, это один аттрибут на все наши сервлеты
        2 - Аттрибуты сессии, это те аттрибуты, которые привязаны к сессии
        3 - Request аттрибуты, это те аттрибуты, которые привязаны к httpServletRequest(к запросам) 
======================================================================================================================================================
Установка аттрибутов - В Спринге, чтобы установить аттрибуты на уровне сессии нужно использовать аннотации @SessionAttribute, @SessionAttributes. 
		       Для установки аттрибутов на уровне запросов используется аннотация @RequestAttribute  

@SessionAttribute    - Используеться в тех случаях, когда нам нужно установить значение в сессии. (На курсе Http/Servlets req.getSession().setAttribute("user", user))
@SessionAttributes   - Используется тогда, когда мы хотим хранить ключи в сессии
@RequestAttribute    - Эта аннотация редко встречается, потому что, все аттрибуты которые мы устанавливаем по умолчиню являются request attribute


================================
Пример: 
Аттрибуты на уровне сессии:   Пользователь залогинился и мы должны привезать его к сессии, на основании этого, мы можем делать фильтрацию на права доступа(Авторизация)
Аттрибуты на уровне запросов: Это когда мы вызываем к примеру у слоя сервис, какой-то метод и возвращаемое значение этого метода мы привязываем к ключу аттрибута, чтобы 
			      в дальнейшенм передать клиенту результат метода по ключу аттрибута
