Liquibase - это инструмент для управления изменениями базы данных, который позволяет отслеживать и контролировать изменения в базе данных.
            С помощью Liquibase можно накатывать скрипты, создавать, изменять и откатывать базы данныхю.

Liquibase - как и все миграционные фреймворки работают под одной схеме. Схема представляет из себя две таблицы
            #1 Таблица: databassechangelog 	<--Содержит информацию по изменению бд
            #2 Таблица: databassechangeloglock  <--Нужен для того, чтобы в один момент времени только один процесс могу накатывать скрипты

Liquibase отправляет запрос на блокировку скриптов, мы сначало ставим pessimistic lock, если lock удался то мы идем в databassechangelog и накатываем новые скрипты.
          Если такие скрипты были то мы их игнорируем.

Liquibase использует распределенную систему блокировки, позволяющую одновременно обновлять базу данных только одному процессу. 
          Другие процессы будут просто ждать, пока блокировка не будет снята.
============================================================================================================================================

Основные понятия:
changelog - скрипт который мы должны накатить на базу данных, он может быть в 4ех разных форматов SQL, XML, JSOM, YAML.
	    changelog состоит из changeset, а changeset состоит из change.

changeset - совокупность change. Один changset в рамках одной транзакции
change    - это одна DML операция

 
===========================================================================================================================================
Хорошим тонном считается использовать один changset в рамках одной транзакции.
Название changelog должен 1. Включать версию 2.Можно добавить дату

Пример: 
changelog-1.0.sql, changelog-1.1.sql, changelog-2.0.sql
