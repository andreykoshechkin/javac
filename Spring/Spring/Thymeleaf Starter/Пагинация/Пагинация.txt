Пагинация - это порядковая нумерация страниц, то есть постраничный вывод данных.Пагинацию часто используют с фильтром. 
Цель пагинации — избежать одномоментной выборки больших объёмов данных
 -Ее внедряют, чтобы:
  -ускорить загрузку страниц;
  -упростить навигацию по сайту;
  -сделать просмотр информации удобным для пользователей;
  -привести дизайн страниц к привычному для пользователей виду.

 -Всего существует две реализации пагинации:
  -Offset пагинация (Встречается чаще всего но имеет проблемы с производительностью) чтобы реализовать offset пагинацию нужно передавать offset/limit.
   offset - какую страницу мы отображаем, limit сколько записей отобразить на странице

  -Keyset пагинация (По мимо limit/offset мы должны передавать ключ, чтобы по этому ключу мы могли искать данные в базе данных)

Spring предоставляет нам Pageable Resolver, который в свою очередь ищет параметры с именами page, size
  это одно из главных условий при работе с пагинацией, следовательно, параметры должны называться строго page, size
 -По умолчанию используеться следующие параметры:
   -Размер page = 0
   -Размер size = 20
